## Upgrading from 0.x

There are no hard breaking changes between the latest 0.x releases and 1.0.0. Semver did not apply pre-1.0, but the public API has been stable since 0.4 and no renames or removals occurred in the 0.4–0.7 window.

The main thing 1.0 adds is a **stability promise**: every exported symbol, Custom Element attribute, theme name, CSS custom property, and UMD global listed in `API.md` is now frozen under semver. Renaming or removing any of them is a breaking change that will require a major version bump.

### Notable facts for 0.x users

**`importIgnores` validation added in 0.6** — Before 0.6 the ignore-list import did no input validation. Since 0.6 it validates that the input is an array, that each entry has a string `ruleId`, and that `selector` (if present) is a string. Malformed data is silently ignored instead of throwing.

**`auto-scan` attribute is a presence attribute** — Setting `auto-scan="false"` in HTML does not disable auto-scan; it must be removed entirely (or set via `mount({ autoScan: false })`). This has been the behavior since 0.1 and is now documented explicitly.

**Default theme is `"auto"`, not `"default"`** — When no `theme` prop/attribute is provided, the resolved theme follows `prefers-color-scheme` and automatically promotes to `"high-contrast"` when `prefers-contrast: more` is set. If you want the dark Catppuccin Mocha colors unconditionally, pass `theme="default"` explicitly.

**`@a11y-hud/solid` uses `createA11yHud`, not `useA11yHud`** — The Solid primitive follows Solid's `create*` naming convention. React, Vue, Angular, and Svelte all export `useA11yHud`; Solid exports `createA11yHud`.

**`@a11y-hud/svelte` hook accepts a getter function** — `useA11yHud` in the Svelte adapter takes `() => UseA11yHudOptions`, not a plain options object. This is required for Svelte 5 `$effect` reactivity.
